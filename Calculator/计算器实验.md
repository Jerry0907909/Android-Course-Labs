# 基于 Android Studio 的简单计算器实现实验报告

## 1. 实验基本信息

### 1.1 实验名称
基于 Android Studio 的简单计算器实现

### 1.2 实验目的
1. 掌握 Android 常用布局控件的综合使用方法，重点理解 `ScrollView`、`LinearLayout`、`GridLayout` 的嵌套布局方式。
2. 掌握 `Button`、`ImageButton`、`TextView` 等常用控件的属性配置与界面设计方法。
3. 理解 Android 按钮点击事件的绑定方式，能够完成数字输入、运算符输入与结果显示的交互逻辑。
4. 熟悉简单计算器中加、减、乘、除、开根号、倒数、清空等功能的实现方法。
5. 提升对界面适配、异常处理与调试分析的实践能力。

### 1.3 实验环境
- 开发工具：Android Studio
- 开发语言：Java
- 构建工具：Gradle
- 测试设备：Android Studio 模拟器 `Small Phone API 37.0`
- 操作系统：macOS

## 2. 实验原理与设计

### 2.1 实验原理
本实验通过 Android 事件驱动模型实现一个简单计算器。用户点击数字按钮后，程序将当前输入内容拼接到字符串中；点击运算符按钮后，程序保存第一个操作数和当前运算符，并在显示区同步展示表达式；点击等号按钮后，程序根据保存的操作数和运算符执行计算，并将结果显示在主显示区中。

整个实现的核心思想是“界面显示”和“运算状态”分离：
- `currentInput` 用于保存当前正在输入的数字；
- `firstOperand` 用于保存第一个操作数；
- `pendingOperator` 用于保存当前等待执行的运算符；
- `expressionView` 用于显示表达式过程；
- `resultView` 用于显示当前输入值或最终计算结果。

### 2.2 关键技术
本实验使用了以下 Android 开发关键技术：

1. **动态文本更新**
   - 使用 `TextView` 实时刷新表达式与结果显示内容。
   - 通过 `updateDisplay()` 方法统一更新界面。

2. **事件监听机制**
   - 使用 `setOnClickListener()` 为各类按钮绑定点击事件。
   - 数字按钮采用统一绑定方式，减少重复代码。

3. **字符串与数值转换**
   - 数字输入以字符串形式拼接。
   - 运算时通过 `Double.parseDouble()` 转换为数值。

4. **格式化输出**
   - 使用 `DecimalFormat` 控制结果显示格式，避免出现过长小数或整数后缀 `.0`。

5. **异常处理**
   - 针对除零、负数开方等非法运算场景，显示 `Error` 提示，防止程序崩溃。

6. **界面优化与分层设计**
   - 通过按钮样式分类实现数字键、功能键、运算键、等号键的差异化设计。
   - 采用双层显示区，增强与 Windows 计算器类似的交互体验。

### 2.3 设计思路
本项目的实现流程如下：

1. 用户点击数字键，更新当前输入字符串；
2. 用户点击运算符，保存第一个操作数，并在表达式区显示“操作数 + 运算符”；
3. 用户继续输入第二个操作数，表达式区同步显示完整表达式；
4. 用户点击 `=`，系统根据前面保存的状态执行计算；
5. 结果显示在大号结果区中，表达式区保留完整计算式；
6. 若用户点击 `CE` 或 `C`，则按照规则清除当前输入或全部状态；
7. 若用户点击 `√` 或 `1/X`，则直接对当前输入值执行一元运算。

### 2.4 界面设计
本项目采用自上而下的界面结构：

- **最外层：`ScrollView`**
  - 用于保证在屏幕空间不足时界面仍可滚动显示。

- **主体布局：`LinearLayout`**
  - 使用垂直方向排列整个计算器内容。

- **显示区域**
  - 采用一个垂直 `LinearLayout` 作为显示面板；
  - 上方小字号 `TextView` 显示当前表达式；
  - 下方大字号 `TextView` 显示当前输入或计算结果。

- **按键区域**
  - 使用 `GridLayout` 构建 5 行 4 列按钮网格；
  - 数字键、功能键、运算键、等号键分别采用不同的样式资源，便于统一管理与后期维护。

该设计既满足实验基础要求，也增强了界面的层次感与可读性。

### 3.2 步骤二：实现显示区双层设计
为了让交互更接近标准计算器，本项目没有只保留一个结果显示框，而是设计为两个文本区域：

- `tv_expression`：显示表达式过程，例如 `12 + 3`
- `tv_result`：显示当前输入值或最终结果，例如 `15`

这样用户在按下 `+` 之后可以立即看到运算符已被记录，提高界面的可理解性。

### 3.3 步骤三：绑定按钮事件
在 `MainActivity.java` 中，项目分别对数字按钮、运算按钮和功能按钮进行绑定。

数字按钮采用统一绑定方式：

```java
private void bindDigitButtons() {
    int[] digitButtonIds = {
            R.id.btn_0, R.id.btn_1, R.id.btn_2, R.id.btn_3, R.id.btn_4,
            R.id.btn_5, R.id.btn_6, R.id.btn_7, R.id.btn_8, R.id.btn_9
    };

    for (int buttonId : digitButtonIds) {
        findViewById(buttonId).setOnClickListener(view ->
                appendDigit(((TextView) view).getText().toString()));
    }
    findViewById(R.id.btn_decimal).setOnClickListener(view -> appendDecimalPoint());
}
```

这种写法避免了为 10 个数字按钮分别编写重复监听代码，结构更清晰。

### 3.4 步骤四：实现输入与表达式显示逻辑
项目中定义了以下关键变量：

```java
private String currentInput = "0";
private double firstOperand = 0D;
private String pendingOperator;
private String expressionText = "0";
private boolean shouldStartNewInput = true;
private boolean lastActionWasEquals = false;
```

它们分别用于保存当前输入、首个操作数、待执行运算符、表达式文本以及输入状态。

当用户点击数字或小数点时，程序会调用 `appendDigit()` 或 `appendDecimalPoint()` 完成输入拼接；当点击运算符时，程序调用 `setOperator()` 保存当前运算状态，并通过 `updateDisplay()` 同步刷新表达式区内容。

表达式构建逻辑如下：

```java
private String buildExpressionText() {
    if ("Error".equals(expressionText)) {
        return expressionText;
    }
    if (pendingOperator == null) {
        return lastActionWasEquals ? expressionText : currentInput;
    }
    if (shouldStartNewInput) {
        return formatNumber(firstOperand) + " " + pendingOperator;
    }
    return formatNumber(firstOperand) + " " + pendingOperator + " " + currentInput;
}
```

该方法使得表达式区可以根据当前状态动态显示：
- 输入数字时显示当前数字；
- 按运算符后显示“首操作数 + 运算符”；
- 输入第二个数字后显示完整表达式；
- 按下等号后保留“表达式 =”的形式。

### 3.5 步骤五：实现运算逻辑
核心运算在 `calculate()` 方法中完成：

```java
private double calculate(double left, double right, String operator) {
    switch (operator) {
        case "+":
            return left + right;
        case "-":
            return left - right;
        case "×":
            return left * right;
        case "÷":
            if (right == 0D) {
                showError();
                return 0D;
            }
            return left / right;
        default:
            return right;
    }
}
```

在点击等号按钮时，程序首先拼接表达式文本，再计算最终结果：

```java
private void evaluateResult() {
    if (pendingOperator == null) {
        updateDisplay();
        return;
    }

    double secondOperand = parseCurrentInput();
    String expressionBeforeResult =
            formatNumber(firstOperand) + " " + pendingOperator + " " + formatNumber(secondOperand) + " =";
    double result = calculate(firstOperand, secondOperand, pendingOperator);
    currentInput = formatNumber(result);
    firstOperand = result;
    pendingOperator = null;
    shouldStartNewInput = true;
    expressionText = expressionBeforeResult;
    lastActionWasEquals = true;
    updateDisplay();
}
```

### 3.6 步骤六：实现功能键
本项目除基础四则运算外，还实现了以下功能：

1. `CE`：清除当前输入
2. `C`：清除所有状态
3. `√`：对当前值求平方根
4. `1/X`：对当前值求倒数
5. `.`：输入小数点

其中一元运算示例：

```java
private void applySquareRoot() {
    double value = parseCurrentInput();
    if (value < 0D) {
        showError();
        return;
    }

    currentInput = formatNumber(Math.sqrt(value));
    shouldStartNewInput = true;
    lastActionWasEquals = false;
    updateDisplay();
}
```

## 5. 问题与解决方案

### 问题1：按钮网格出现视觉不对齐
**问题现象：**
初期布局中，`1/X` 按钮因为文本较长，在部分显示比例下出现换行，导致该按钮所在格子看起来没有与其他按钮对齐。

**原因分析：**
按钮默认存在字体留白与内部边距，而 `1/X` 文本长度相对更长，导致视觉高度和内容重心与其他按钮不一致。

**解决方案：**
1. 将按钮样式统一抽取到 `styles.xml` 中；
2. 设置按钮单行显示；
3. 去除多余内边距与字体留白；
4. 适当减小 `1/X` 按钮字号。

**解决效果：**
按钮网格整体对齐，界面更统一。

### 问题2：按下运算符后无法直观看到表达式
**问题现象：**
早期实现中，显示框只显示当前输入或结果。当用户按下 `+`、`-` 等按钮后，界面上无法明确看出当前运算状态。

**原因分析：**
单一显示框只能展示一个数值，不能同时表达“已输入的第一个数”和“待执行运算符”。

**解决方案：**
1. 将显示区改为双层结构；
2. 上层显示表达式过程；
3. 下层显示当前输入值或最终结果；
4. 编写 `buildExpressionText()` 动态拼接表达式。

**解决效果：**
按下 `+` 之后可以立刻显示 `12 +`，继续输入后显示 `12 + 3`，交互更接近标准计算器。

### 问题3：非法运算可能导致程序逻辑异常
**问题现象：**
例如进行除零或对负数开方时，如果不做限制，结果可能异常，影响用户体验。

**原因分析：**
这类操作属于数学上的非法输入，需要在程序层面主动判断。

**解决方案：**
1. 在除法中判断除数是否为 0；
2. 在开方中判断输入值是否小于 0；
3. 出现非法情况时调用 `showError()`；
4. 将表达式区和结果区都更新为错误状态，并重置内部变量。

**解决效果：**
程序不会崩溃，用户能够及时获得明确反馈。
